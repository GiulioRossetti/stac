Descripción

Los tests no paramétricos son aquellos que no cumplen alguna de las condiciones establecidas para los tests paramétricos:

  • Normalidad: Las muestras siguen una distribución normal.
  • Independencia: Las muestras no dependen de otras.
  • Homocedasticidad: Igualdad de varianzas en las muestras.

Test de Wilcoxon

Contrasta la hipótesis nula de que las medianas de las diferencias de dos muestras (resultados de dos algoritmos) relacionadas son iguales. Para tamaños muestrales pequeños, se puede determinar mediante la comparación del estadístico con el valor crítico de la tabla de Wilcoxon (si estadistico < valor crítico se rechaza H0). Para tamaños muestrales grandes (> 25), el test se puede aproximar con la distribución normal (en cuyo caso H0 sería rechazada si p-valor < alpha).

Restricciones:

  • Número de algortimos >= 2
  • Se deben tener al menos 5 conjuntos de datos cuyas diferencias sean distintas de 0.

Los tests no paramétricos de ranking, contrastan la hipótesis nula de que todos los algoritmos se comportan de forma similar, en cuyo caso sus rankings deberían de ser similares y por tanto se podrían considerar iguales todos los algoritmos. Los tests de ranking se diferencian en la forma de calcular los rankings o en la forma de calcular el estadístico.

Estos tests, dan como resultado la existencia o no de diferencias significativas entre los algoritmos sobre los que se han aplicado. Es decir, nos dice si el contraste de hipótesis es o no estadísticamente significativo. Si se rechaza la hipótesis nula de “todos los algoritmos son iguales”, sabremos que entre los algoritmos existen diferencias. Sin embargo, puede ocurrir que un algoritmo presente un rendimiento similar a otro u otros y por tanto se pueda considerar igual.

Por este motivo, después de aplicar los tests de ranking se aplican los tests Post-Hoc. Estos tests comparan los algoritmos y realizan contrastes de hipótesis entre ellos para determinar diferencias. Se distinguen dos tipos de comparación:

  • Método de control: Se compara el primer algoritmo del ranking devuelto por el test de ranking con el resto de algoritmos y por tanto habrá K-1 comparaciones.
  • Comparación múltiple (multitest): Compara todos los algoritmos con todos. Habrá K*(K-1)/2 comparaciones.

Selecciona un test de ranking:

Friedman: Realiza comparaciones y asigna rankings dentro de cada conjunto de datos. El estadístico se distribuye como una distribución chi-cuadrado con K-1 grados de libertad, siendo K el número de variables relacionadas (o número de algoritmos).

Iman-Davenport: Mejora del test de Friedman con un estadístico más ajustado, que se distribuye de acuerdo a una distribución f con (K-1) y (K-1)*(N-1) grados de libertad, siendo N el número de conjunto de datos.

Rangos Alineados de Friedman: Realiza comparaciones y asigna rankings teniendo en cuenta a todos los conjuntos de datos. Suele emplearse cuando el número de algoritmos en la comparación es pequeño.

Quade: Similar a Iman-Davenport, pero considera que algunos problemas son más difíciles o que los resultados que obtienen los algoritmos sobre ellos son más distantes (ponderación).

Selecciona el test Post-Hoc a aplicar:

Por defecto se aplica Bonferroni-Dunn

Bonferroni-Dunn: Rechaza la hipótesis nula si: p-valor < alpha / (K-1)

Holm: Compara cada pi (empezando por el más significativo o pequeño) con alpha / (K−i), donde i [1,K-1]. Si se rechaza una hipótesis continúa contrastando. En el caso de que una hipótesis se acepte se aceptan todas las demás.

Hochberg: Compara en la dirección opuesta a Holm. En el momento que encuentra una hipótesis que pueda aceptar, acepta todas las demás.

Li: Rechaza todas las hipótesis si el p-valor menos significativo es menor que alpha. En otro caso, acepta dicha hipótesis y rechaza cualquier hipótesis restante cuyo p-valor sea menor que un valor calculado.

Finner: Sigue un proceso igual al test de Holm pero cada p-valor asociado con la hipótesis Hi se compara como: pi <= alpha / 1−(1−alpha)^(K−1)/i, donde i [1,K-1]

Multitests: Iguales que los demás pero en lugar de K-1 comparaciones se relizan m donde, m=(K∗(K−1))/2.

Shaffer: Igual a Holm pero cada p-valor asociado con la hipótesis Hi se compara como pi <= alpha / ti, donde ti es el número máximo de hipótesis posibles suponiendo que las (j-1) anteriores han sido rechazadas.
